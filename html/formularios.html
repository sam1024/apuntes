Formularios

LOS FORMULARIOS SIRVEN PARA ENVIAR DATA DE PARTE DEL USUARIO, PARA HACER UN FORMULARIO SE USA LA ETIQUETA form, ESTA CUENTA CON LOS ATRIBUTOS:
action: INDICA QUE ARCHIVO VA A PROCES ESTE FORMULARIO.
method: INDICA LA MANERA EN COMO SE ENVIARAN LOS DATOS (POST O GET)
LOS FOMULARIOS ESTÁN COMPUESTOS POR CAMPOS, PARA PINTAR UN CAMPO SE USA LA ETIQUETA input ESTA TIENE VARIOS ATRIBUTOS, ENTRE LOS MAS USADOS ESTAN:.
type: INDICA EL TIPO DE CAMPO, PUEDE SER text, number, password, hidden, button, email, date, submit, check, radio, select etc.
name: INDICA EL NOMBRE DEL CAMPO, ESTE ATRIBUTO ES INDISPENSABLE YA QUE PERMITIRÁ RECIBIR LOS DATOS EN EL ARCHIVO QUE TENGAMOS COMO ACTION.
value: INDICA EL VALOR POR DEFECTO DEL CAMPO, EN CASO DE SER DE type BUTTON EL VALUE SERÁ EL TEXTO QUE APARECERÁ EN EL BOTÓN.
placeholder: MUESTRA UN TEXTO DENTRO DEL CAMPO, ESTE TEXTO AUTOMATICAMENTE SE BORRA CUANDO EL USUARIO COMIENZA A ESCRIBIR.
required: OBLIGA AL USUARIO A LLENAR EL CAMPO DE TEXTO CON LA INFORMACIÓN.
readonly: MUESTRA UN VALOR POR DEFECTO EN UN CAMPO, ESTE NO PUEDE SER MODIFICADO POR EL USUARIO.

CODIGO FUENTE DE UN FORMULARIO BÁSICO PARA LOGUEO

                <form action="archivo.php" method="POST">
                  <label for="user">Usuario<label>
                  <input type="text" name="user" id="user">
                  <label for="pass">Contraseña<label>
                  <input type="password" name="pass" id="pass">
                  <input type="submit" name="send" value="Login">
                </form>

Tipos de inputs

AUNQUE HTML5 OFRECE EL NUEVO ATRIBUTO required PARA LOS TIPOS DE INPUTS QUE ES UTIL HASTA CIERTO PUNTO PARA VALIDAR QUE EL USUARIO INGRESE DATOS CORRECTOS ESTE ES DEMASIADO FACIL DE BURLAR.
EJEMPLO: TENEMOS UN INPUT DE TIPO MAIL Y SI EL USUARIO PONE UN TEXTO CUALQUIERA, AL MOMENTO DE TRATAR DE ENVIAR LA INFORMACIÓN EL NAVEGADOR MOSTRARÁ UN MENSAJE DE QUE LOS DATOS DEL CAMPO NO ES UNA DIRECCIÓN DE CORREO VALIDA Y LO MISMO PASARA CON LOS DEMÁS TIPOS DE INPUT SIEMPRE Y CUANDO USEMOS EL ATRIBUTO required.

CODIGO FUENTE DE UN FORMULARIO USANDO EL ATRIBUTO required PARA LOGUEO

                <form action="archivo.php" method="POST">
                  <label for="user">Usuario<label>
                  <input type="text" name="user" id="user" required>
                  <label for="pass">Contraseña<label>
                  <input type="password" name="pass" id="pass" required>
                  <input type="submit" name="send" value="Login">
                </form>

EJEMPLO DE UN FORMULARIO USANDO EL ATRIBUTO required PARA LOGUEO

Campos de selección y atributos

AL MOMENTO DE DISEÑAR UN FORMULARIO TENEMOS QUE PENSAR EN LA EXPERIENCIE DEL USUARIO (UX), NO DEBEMOS PONER DEMASIADOS CAMPOS QUE LO ABRUMEN Y ESTE DEJE DE VISITAR NUESTRA PAGINA O DEJE DE USAR NUESTRA APP.

EJEMPLO DE UN FORMULARIO CON MUCHOS CAMPOS



VER UN FORMULARIO COMO ESTE A MAS DE UNO LE DARÍA FLOJERA LLENARLO Y PROBABLEMENTE NO SE REGISTRARÍA, ENTONCES PARA EVITAR ESTE TIPO DE SITUACIONES DEBIDO A QUE AVECES ES NECESARIO TENER MUCHOS CAMPOS DE TEXTOS, EXISTE UN ELEMENTO LLAMADO fieldset, ESTE PERMITE METER VARIOS CAMPOS EN UNA ESPECIE DE CONTENEDOR.

CÓDIGO FUENRE DE EJEMPLO DE ELEMENTO fieldset

            <fieldset>
             <legend>Titulo del fieldset</legend>
            </fieldset>
